# v1.0.0 — Congelar

Este release marca o fim da janela em que mudanças breaking são livres.
Pós-1.0, `outbox-core` é sagrado: zero breaking change. Evolução só via
`default` methods, novos tipos, novos módulos opt-in.

Cada item abaixo é gate; se algum não estiver verde, **não release**.

---

**Branch:** `chore/v1-spi-audit`
**Commit prefix:** `chore:`

**Tarefa.**

1. Listar todo tipo `public` em `outbox-core` e `outbox-jdbc.spi`.
2. Para cada um, confirmar:
   - Javadoc completa em todo método/construtor público.
   - `@since` correto.
   - Não há tipo retornado/parâmetro que vaze classe `internal`.

4. Gerar relatório no PR description com a lista final dos tipos que
   serão congelados.

**Aceitação.**

- Lista revisada por humano antes do merge.
- Javadoc CI verde.

## Prompt 5 — ADR de política pós-1.0

**Branch:** `docs/adr-v1-policies`
**Commit prefix:** `docs:`

**Tarefa.** Criar três ADRs:

1. `docs/adr/0021-api-stability-post-1.0.md`:
   - `outbox-core` API: zero breaking change pós-1.0.
   - `outbox-jdbc` API: zero breaking change pós-1.0.
   - SPI (`outbox-jdbc.spi`): evolução só via `default` methods.
   - Decorators (`outbox-micrometer`, `outbox-otel`): podem evoluir com
     bump major próprio se necessário.
   - `outbox-spring`: segue Spring Boot's compat policy.
2. `docs/adr/0022-deprecation-policy.md`:
   - Métodos deprecated marcados com `@Deprecated(forRemoval=true, since="X.Y")`.
   - Mínimo de 2 minor releases antes da remoção.
   - Remoções só em major bump (`2.0.0`).
3. `docs/adr/0023-jdk-floor-post-1.0.md`:
   - Floor JDK 25 LTS congelado em 1.x.
   - Bump de floor só em major bump.
   - Suporte a JDK 17/21 retroativo: **não**.

**Aceitação.**

- ADRs mergeadas, referenciadas de `CONTRIBUTING.md`.
